### 1.1.1 编译执行与解释执行
- 编译执行：运行编译器对源代码进行编译，编译的过程并不执行程序，而是把源代码全部翻译成机器指令，再加上一些描述信息，生成可执行文件，可执行文件可以被操作系统加载运行
- 解释执行：由解释器逐句解析源代码并即时转换为底层指令执行，无需生成独立的目标程序，执行结果通过终端实时反馈。该方式通过虚拟机或解释器实现跨平台特性，使得同一份代码可在不同操作系统的运行时环境中运行。其特点包括开发效率较高且错误定位精确，但运行效率和资源利用率低于编译执行。

### 1.4.1 gcc编译基本命令
```
$ gcc main.c -o main
```
上述代码表示：使用gcc编译main.c源代码，生成名为main.out的可执行文件
```
$ gcc -Wall main.c
```
上述代码表示：打开gcc的-Wall选项，提示所有的警告信息，推荐每个命令都加入-Wall选项
### 3.2.1 main函数
main函数最标准的写法应该如下所示：
```
int main(int argc, char *argv[])
```
也允许：
```
int main(void)
```
除这两种之外，其他写法都是错误的或不可移植的
### 3.3.1 Man page
Man Page是Linux开发最常用的参考手册，由很多页面组成，每个页面描述一个主题，这些页面被组织成若干个Section。FHS（Filesystem Hierarchy Standard）标准规定了Man Page各Section的含义如下：

| Seciton | 描述                                                |
| ------- | ------------------------------------------------- |
| 1       | 用户命令，例如`ls(1)`                                    |
| 2       | 系统调用，例如`_exit(2)`                                 |
| 3       | 库函数，例如`printf(3)`                                 |
| 4       | 特殊文件，例如`null(4)`描述了设备文件`/dev/null`、`/dev/zero`的作用 |
| 5       | 系统配置文件的格式，例如`passwd(5)`描述了系统配置文件`/etc/passwd`的格式  |
| 6       | 游戏                                                |
| 7       | 其它杂项，例如`bash-builtins(7)`描述了`bash`的各种内建命令         |
| 8       | 系统管理命令，例如`ifconfig(8)`                            |
