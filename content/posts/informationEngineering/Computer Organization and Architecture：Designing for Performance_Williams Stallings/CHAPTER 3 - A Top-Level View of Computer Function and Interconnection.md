# 3.1 Computer function
- **Basic function:** Execution program
	- **Program:** A set of instructions stored in memory
The program execution consists of repeating the ***instruction cycle***.
## 3.1.1 Instruction cycle
means the processing required for a **single** instruction
![](attachments/fetch%20cycle.png)
as shown in above fig, it consists of:
- Fetch cycle
- Execute cycle
### 3.1.1.1 Fetch cycle
- Program Counter (PC) ***holds address*** of next instruction to fetch
- Processor ***fetches instruction*** from memory location pointed to by PC
- ***Increment PC*** after each fetch (Unless told otherwise)
- ***Instruction loaded*** into Instruction Register (IR)
### 3.1.1.2 Execute cycle
Processor interprets instruction and performs required actions
- Processor-memory
	- Data transfer between CPU and main memory
- Processor-I/O
	- Data transfer between CPU and I/O module
- Data processing
	- Some arithmetic or logical operation on data
- Control
	- Alteration of sequence of execution (e.g. jump)
- Combination of above
> Alteration: n. 改变
# 3.2 Interrupts
Mechanism by which other modules may interrupt normal sequence of processing.
- Interruption of the normal sequence of execution
- ***No changes*** are needed in the user program
- Processor and operating system are responsible for interrupt

Most common classes of interrupts:
- Program
	- overflow
	- division by zero
	- illegal instruction
	- outside of the memory
	- ...
- Timer: Generated by internal processor timer, to perform certain functions *on a regular basis*
- I/O: from I/O controller
- Hardware failure
	- memory parity error
	- ...

> on a regular basis: 定期

## 3.2.1 Transfer of Control via Interrupts
![](attachments/transfer%20via%20interrupts.png)
> resume: v. (中断后)继续

The interrupt cycle is added to instruction cycle,  shown in the following picture in detail:
![[attachments/interrupt cycle.png]]
If no interrupt:
- Fetch next instruction

If interrupt pending:
1. Suspend execution of current program and save its context
2. Set PC to start address of interrupt handler routine
3. Process interrupt
4. Restore context and continue interrupted program
## 3.2.2 Multiple Interrupts
> [!NOTE]
> The priorities allow interrupt with higher priority to cause a lower-priority interrupt handler to be interrupted
### 3.2.2.1 Disabled interrupt
![](attachments/disabled%20interrupt.png)
As shown in the figure above, processor can and will ignore the interrupt request signal.
If an interrupt occurs, it remains pending and will be checked after the processor has enabled interrupts.
### 3.2.2.2 Interrupt with priorities
People can define priorities for interrupts request signal, and allow interrupt with higher priority to cause a lower-priority interrupt handler to be interrupted.

As shown in the figure below, there are three I/O devices: printer, disk, communication line.
![](attachments/priority%20interrupt.png)
Because the disk interrupt occurs at t=20 but is processed at t=25, the priority of the disk is lower than communication interrupt. And the both are higher than printer interrupt.
# 3.-1 Key Points
An instruction cycle consists of:
- an instruction fetch
- followed by zero or more operand fetches
- followed by zero or more operand stores
- followed by an interrupt check (if interrupts are enabled).
